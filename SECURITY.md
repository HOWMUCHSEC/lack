# Security Policy

## Supported Versions

Security fixes are provided for the current `main` branch until formal release branches are published.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately before public disclosure.

- Email: security@lack-lab.com
- Include affected version/commit, reproduction steps, impact, and any proof-of-concept details.
- Do not include secrets, production customer data, or unauthorized scan results in reports.

We aim to acknowledge reports within 5 business days. Confirmed issues will be triaged by severity and fixed as quickly as practical.

## Authorized Testing

Lack includes active security scanning features. Only scan systems you own or have explicit permission to test. You are responsible for complying with all applicable laws, terms of service, and network policies.

## Telemetry

Open-source builds do not send Sentry telemetry unless a Sentry DSN is explicitly configured.
